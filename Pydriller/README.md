# PyDriller Repository Extraction Environment

This folder serves as the central data-lake containing massive-scale historical repository metadata extracted from major open source hubs. These raw records function as our baseline control variables (representing humanly generated `0` code markers) when training and calibrating our Natural Language LLM detectors further down the workflow stream.

## Extraction Scripting
- `extract_airflow_data.py`: A native Python script deploying the PyDriller library module. When executed, it clones massive established repository trees locally (e.g., Apache Airflow, Django) and traverses their historical commit tree recursively.
  - It extracts critical data structures directly from the nodes: Commit Messages, Insertions/Deletions, Filenames, Code Complexity (NLOC, Token Counters), and absolute Source Code changes representing what humans altered at massive scale.
  - Generates terabytes of contextually accurate, non-AI tainted `.JSON` and `.CSV` metric sheets.

## The Data Repositories

**Note:** For structural safety on Github pushes, all data-files with extensions (`.json`, `.csv`, `.xlsx`, `.parquet`) generated out of this directory **are excluded from Git version control limits natively via the `.gitignore` module**. They operate purely on your local file-system to protect your repository from GitHub's strict 100MB object caps.

### Parquet Datasets (`*.parquet`)
- **`commit_metrics.parquet` (Main Driver)**: Validated history rows utilized by our active `DistilBERT` arrays. Maps 5,000+ developer commits directly supplying message strings as Human (`0`) tokens.
- **`hunk_metrics.parquet`**: Narrowly mapped text-hunks tracking character bounds and method-level line replacements originally generated by legacy testing pipelines.

### Legacy Raw Exports
