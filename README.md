# AIvsHuman: Source Code Authenticity Detection

This project evaluates and compares different methodologies for distinguishing between human-written code and AI-generated code. The research focuses on analyzing GitHub repositories, specifically using the **Apache Airflow** repository as a primary source for human-authored code.

## 📂 Repository Structure

The following components perform distinct roles in the detection and analysis pipeline:

| Folder / File / Link | Role | Description |
| :--- | :--- | :--- |
| `Pydriller/` | **Data Miner** | Uses the PyDriller framework to analyze Git history and extract code modifications from commits. |
| `GPTSniffer/` | **Classifier** | A tool used to categorize code snippets as either human or ChatGPT-generated using specialized datasets. |
| [GPTSniffer Datasets (GitHub)](https://github.com/MDEGroup/GPTSniffer/tree/master/DATASETS) | **Evaluation Baseline** | Ground-truth datasets containing human and ChatGPT-generated code pairs used for model benchmarking. |
| `detect-gpt-fork/` | **Zero-Shot Auditor** | A repository integrated into this project to identify AI code through probability analysis. |
| [PyDriller Datasets (Drive)](https://drive.google.com/drive/folders/106hl1dZu21w-5GcNaiWWDC8FiLtKw1Lu) | **Cloud Data Storage** | A dedicated Google Drive folder containing raw commit history extracted using PyDriller. |
| `airflow_commits_data.json` | **Local Dataset** | The local output generated by PyDriller, containing extracted code changes from Apache Airflow. |

## 🚀 Setup & Installation

### 1. Requirements
The project is built using several core languages and frameworks:
* **Python & PyTorch**: Essential for the deep learning models and data processing scripts.
* **PyDriller**: Required for mining commit data and code churn from Git repositories.
* **CodeBERT/Transformers**: Used for the classification tasks within the GPTSniffer component.

### 2. Environment Setup
1.  **Virtual Environment**: It is recommended to use a Python virtual environment to manage project-specific dependencies.
2.  **Dependencies**: Install required packages, including `pydriller`, `torch`, and `transformers`.

---

## 🛠 Running the Pipeline

### Phase 1: Data Extraction
* **Action**: Use the scripts within the `Pydriller/` directory to mine the Apache Airflow repository.
* **Storage**: Extracted datasets are synced to the [Cloud Data Storage](https://drive.google.com/drive/folders/106hl1dZu21w-5GcNaiWWDC8FiLtKw1Lu) folder for accessibility and backup.

### Phase 2: AI vs. Human Classification
* **Action**: Run the `GPTSniffer` classifier against your extracted data.
* **Reference**: Compare the results from your local `airflow_commits_data.json` against the [GPTSniffer Datasets](https://github.com/MDEGroup/GPTSniffer/tree/master/DATASETS) to validate the model's accuracy on known human vs. AI samples.

### Phase 3: Zero-Shot Verification
* **Action**: Use the `detect-gpt-fork` component to analyze snippets without specific training.
* **Method**: This analyzes the "probability curvature" of code snippets to detect machine-generated patterns.

---

## 📊 Summary of Analysis
This project addresses the challenge of code authenticity by comparing results from **GPTSniffer** and **DetectGPT**. By using verified human commits from a large-scale project like **Apache Airflow**, you can evaluate how effectively AI-generated code patterns can be identified in real-world development environments.
