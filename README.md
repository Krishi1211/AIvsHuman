# AIvsHuman: Source Code Authenticity Detection

This project evaluates and compares different methodologies for distinguishing between human-written code and AI-generated code. The research focuses on analyzing GitHub repositories, specifically using the **Apache Airflow** repository as a primary source for human-authored code.

## 📂 Repository Structure

Based on the current project files, the following components perform distinct roles in the detection pipeline:

| Folder / File | Role | Description |
| :--- | :--- | :--- |
| `Pydriller/` | **Data Miner** | Uses the PyDriller framework to analyze Git history and extract code modifications from commits. |
| `GPTSniffer/` | **Classifier** | A tool used to categorize code snippets as either human or ChatGPT-generated using specialized datasets. |
| `detect-gpt-fork/` | **Zero-Shot Auditor** | A repository integrated into this project to identify AI code through probability analysis. |
| `airflow_commits_data.json` | **Raw Dataset** | The output generated by PyDriller, containing the extracted code changes from Apache Airflow. |

## 🛠 Running the Pipeline

### Phase 1: Data Extraction
* **Action**: Use the scripts within the `Pydriller/` directory to mine the Apache Airflow repository.
* **Result**: This step generates the `airflow_commits_data.json` file, which serves as your human-written baseline.

### Phase 2: AI vs. Human Classification
* **Action**: Run the `GPTSniffer` classifier against your extracted data.
* **Data**: Reference the datasets provided in the `GPTSniffer/DATASETS` directory to validate the model's accuracy.

### Phase 3: Zero-Shot Verification
* **Action**: Use the `detect-gpt-fork` component to analyze the snippets without specific training.
* **Method**: This analyzes the "probability curvature" of code snippets to detect machine-generated patterns.
